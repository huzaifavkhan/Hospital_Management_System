WORKFLOW 1: Patient Management
--------------------------------------------
1. User (receptionist) opens the application and logs in.
2. User navigates to the "Patient Management" section.
3. User clicks "Add New Patient".
4. A form appears asking for patient details: name, age, gender,
   contact number, address, and medical history.
5. User fills in the details and clicks "Register Patient".
6. System validates all fields for completeness and correct format.
7. If valid, patient record is saved and a unique Patient ID is generated.
8. User can search for existing patients using name or Patient ID.
9. User can view, update, or delete any patient record.
10. System displays a confirmation message after every action.

WORKFLOW 3: Administrative Control
--------------------------------------------
1. Admin logs into the system using admin credentials.
2. Admin navigates to the "Administrative Control" dashboard.
3. Admin can manage departments: add, update, or remove departments.
4. Admin can view overall hospital statistics such as total patients,
   total doctors, and department-wise breakdown.
5. Admin can manage user accounts (receptionist, staff) including
   creating, editing, and deactivating accounts.
6. Admin can generate reports on patient and doctor activity.
7. Admin can set system-wide settings such as hospital name and contact info.
8. All actions performed by the admin are logged for audit purposes.
9. System displays confirmation or error messages for every action taken.
